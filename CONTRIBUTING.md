# CONTRIBUTING.md

## Principles
- Keep scope aligned with MVP docs
- Do not introduce framework churn
- Favor readable server-side code
- Keep templates modular and partial-friendly
- Write tests for every queue rule change

## Branching guidance
- Small coherent feature branches
- One major workflow per branch when practical
- Update docs when scope or implementation meaningfully changes

## Code review focus
- Does it respect the locked stack?
- Does it preserve queue correctness?
- Does it keep patient history intact?
- Does it avoid unnecessary complexity?
