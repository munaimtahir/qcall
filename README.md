# Queue Management MVP — AI Developer Pack

This pack is the source-of-truth starter set for building a **single-doctor OPD queue management web application with PWA support**.

## Locked stack
- Backend: Django
- Database: PostgreSQL
- Frontend: Django Templates + HTMX + Tailwind CSS
- JS: Minimal Alpine.js only where needed
- Deployment: Docker + Caddy
- Product scope: Single-doctor OPD queue + lightweight patient history

## What this pack contains
- Product description and locked scope
- Architecture and module breakdown
- Data model and queue rules
- API and URL interface plan
- Build stages and task checklist
- QA and test acceptance criteria
- Setup and deployment skeleton
- Final autonomous AI developer prompt
- Initial repository scaffold

## Core product definition
Build a **single-doctor OPD queue management system** that allows reception to register patients, search returning patients, generate daily queue tokens, manage live queue flow, let the doctor call/complete/skip patients, preserve lightweight visit history, and show a public display screen.

Appointments are explicitly excluded from MVP.
